Thoughts on Progressive Enhancement

Progressive enhancement is a web design strategy that puts emphasis on web content first and foremost, loading a page’s text through HTML source code rather than having to wait for JavaScript to initiate and load the content – a process that modern parlance has nicknamed “graceful degradation”. This enables a web page to load faster, and also renders it backward compatible for users with slower connections. Users with faster connections still get better loading times, but this “one size fits all” approach still ultimately enables all users to view web content, whereas one JavaScript error would render a site unreadable.

Graceful degradation focused on building websites for the most advanced, capable browsers. The emphasis was on content, and there was very much an Excelsior (forever forward) mindset to it, as the early niche market of tech-literate users were expected to keep their equipment up to date. However, this philosophy clashed violently with the vision of a universally accessible web championed by the creator of HTML itself:

“For anyone to be able to publish anything on the web, all the computers involved have to speak the same languages to each other, no matter what different hardware people are using; where they live; or what cultural and political beliefs they have. In this way, the web breaks down silos while still allowing diversity to flourish.”
-Sir Timothy Berners-Lee, World Wide Web Consortium Univerality Policy

Progressive Enhancement is based on six core principles:

  • Basic content should be accessible to all web browsers.
  • Basic functionality should be accessible to all web browsers.
  • Sparse, semantic markup contains all content.
  • Enhanced layout is provided by externally linked CSS.
  • Enhanced behavior is provided by externally linked JavaScript.
  • End-user web browser preferences are respected.
